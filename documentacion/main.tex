\section{Informe de guía práctica}

\subsection{Objetivos}

\subsubsection{General}
Desarrollar un sistema web de facturación para una ferretería, utilizando React, ASP.NET Core, SQL Server y Entity Framework Core, 
bajo una arquitectura orientada a microservicios con dos bases de datos separadas para clientes y procesos.

\section{Objetivos específicos}
\begin{enumerate}
\item Diseñar una arquitectura basada en dos microservicios.
\item Implementar base de datos de clientes.
\item Implementar base de datos de procesos.
\item Permitir búsqueda de clientes.
\item Registrar nuevos clientes.
\item Construir flujo de facturación.
\item Actualizar inventario.
\item Documentar el sistema.
\end{enumerate}

\subsection{Instrucciones}
El proyecto consiste en el desarrollo de un sistema web orientado a la simulación del proceso de facturación de una ferretería. La aplicación permitirá registrar clientes, consultar su información, gestionar productos, construir una factura a partir de varios ítems y actualizar el stock de inventario una vez emitida la transacción.
La solución será implementada con una arquitectura orientada a microservicios, utilizando React para el frontend, ASP.NET Core Web API para el backend, SQL Server como sistema gestor de bases de datos y Entity Framework Core como ORM para el acceso y manipulación de los datos. Con el fin de evidenciar el uso de microservicios, la información será dividida en dos bases de datos independientes: una destinada al manejo de clientes y otra al proceso de productos y facturación.
La propuesta contempla, además, distintos escenarios de identificación del comprador. El sistema permitirá facturar como consumidor final o con datos del cliente, considerando tanto personas naturales como personas jurídicas. La emisión de la factura se realizará en función del tipo de identificación entregado al momento de la compra.

\section{Título del proyecto}
Sistema web de facturación para una ferretería, basado en arquitectura de microservicios, utilizando React, ASP.NET Core, SQL Server y Entity Framework Core.

\section{Descripción general del proyecto}
El proyecto consiste en el desarrollo de un sistema web orientado a la simulación del proceso de facturación de una ferretería. La aplicación permitirá registrar clientes, consultar su información, gestionar productos, construir una factura a partir de varios ítems y actualizar el stock de inventario una vez emitida la transacción.

La solución será implementada con una arquitectura orientada a microservicios, utilizando React para el frontend, ASP.NET Core Web API para el backend, SQL Server como sistema gestor de bases de datos y Entity Framework Core como ORM.

La información será dividida en dos bases de datos independientes: una destinada al manejo de clientes y otra al proceso de productos y facturación.

El sistema permitirá facturar como consumidor final o con datos del cliente, considerando tanto personas naturales como jurídicas.

\section{Justificación}
El desarrollo del sistema responde a la necesidad de aplicar de forma práctica los contenidos estudiados en la asignatura, integrando tecnologías actuales de desarrollo web, arquitectura de software y persistencia de datos.

El uso de microservicios permite separar responsabilidades y mejorar la organización del sistema. Además, Entity Framework Core facilita el trabajo con un enfoque orientado a objetos.

El sistema sigue un flujo claro: identificar cliente, seleccionar productos, validar stock, calcular valores y emitir la factura.

\section{Alcance del proyecto}
El sistema permitirá:
\begin{itemize}
\item Emitir facturas para consumidor final.
\item Emitir facturas con clientes registrados.
\item Registrar clientes naturales y jurídicos.
\item Buscar clientes por identificación.
\item Registrar productos.
\item Agregar productos a facturas.
\item Calcular valores.
\item Descontar stock.
\end{itemize}

No incluirá integración con SRI ni pagos electrónicos.

\section{Flujo de facturación}
\begin{enumerate}
\item El usuario abre Nueva Factura y elige tipo de facturación.
\item Si selecciona consumidor final, se usa un cliente genérico.
\item Si selecciona cliente, se busca por identificación o se registra.
\item Selecciona productos y arma el detalle.
\item El backend valida stock y calcula subtotal, descuento, IVA y total.
\item Se emite la factura: guarda cabecera, detalle y descuenta stock.
\end{enumerate}

\subsection{Conclusiones}

La presente propuesta plantea una solución ordenada, viable y coherente con los requerimientos del proyecto académico. La separación en microservicios, el uso de dos bases de datos independientes y la implementación con React, ASP.NET Core, SQL Server y Entity Framework Core permiten demostrar conocimientos tanto de desarrollo web como de arquitectura de software.
Además, la estructura definida para clientes, productos y facturación facilita el manejo de distintos escenarios de venta, manteniendo un diseño de base de datos claro y adaptable. El proyecto, por tanto, no solo responde al requisito técnico planteado, sino que también constituye una propuesta sólida y defendible para su presentación académica.

\subsection{Recomendaciones}
Se recomienda verificar cuidadosamente la coherencia entre el modelo relacional y el modelo físico antes de crear las tablas en Oracle, con el fin de evitar errores en claves, nombres de campos o relaciones entre entidades. También es conveniente ingresar datos de prueba que representen distintos escenarios, ya que ello facilita la comprobación del correcto funcionamiento de las restricciones y relaciones definidas.

Además, se recomienda continuar practicando consultas DQL y subconsultas sobre las bases de datos implementadas, puesto que esta fase permite interpretar la información almacenada y validar de manera más completa el trabajo realizado en la construcción del esquema físico.
